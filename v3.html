18
SETUP EXAMPLE

실제 셋팅은 이렇게

CLAUDE.md

# 프로젝트 규칙
- 언어: TypeScript
- 테스트: 80% 커버리지 필수
- 커밋: conventional commits
- 보안: API키 하드코딩 금지

Hooks (settings.json)

"PostToolUse": [{
  "matcher": "Write|Edit",
  "command": "eslint --fix $FILE"
}]
→ 코드 수정마다 자동 린트

Agents (병렬 실행)

Agent 1: code-reviewer
Agent 2: security-reviewer
Agent 3: tdd-guide
→ 3개가 동시에 코드 검증

Memory

memory/
├── user_role.md
├── feedback_testing.md
└── project_goals.md
→ 세션이 바뀌어도 기억 유지

이 4가지만 셋팅하면, AI가 팀원처럼 일한다

HOW TO USE

코드 없이, 말로 운용한다

리서치 요청

"이거 에이전트 소환해서 리서치해줘"

→ AI가 알아서 탐색 에이전트를 띄워 조사 후 결과 보고

병렬 검증

"보안 검토랑 코드 리뷰 동시에 해줘"

→ security-reviewer + code-reviewer 2개 에이전트 병렬 실행

자동 배포

"GitHub Pages로 배포해줘"

→ 레포 생성, 파일 push, Pages 활성화까지 자동 완료

이 프레젤테이션

"해커톤 발표자료 만들어줘"

→ 이 프레젤테이션 전체가 자연어 명령으로 만들어졌습니다

프로그래밍 언어가 아닌, 자연어가 새로운 인터페이스

/plan 기능 구현 전 설계

AI:  → PRD 작성 → 아키텍처 설계 → 태스크 분해 → 구현 순서 제안

you: /tdd 결제 금액 계산 함수
AI:  → 테스트 작성(RED) → 구현(GREEN) → 리팩토링 → 커버리지 80%+ 확인
/security-scan 보안 취약점 점검 
you: /security-scan
AI:  → OWASP Top 10 점검 → SQL 인젝션 탐지 → API키 노출 확인 → 리포트
/review-pr PR 코드 리뷰 
you: /review-pr 42
AI:  → diff 분석 → 버그 탐지 → 성능 이슈 → 개선안 제시 → 승인/반려
